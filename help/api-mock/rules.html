Mock 语法

Apifox Mock 语法完全兼容 Faker.js 和 Mock.js，并且还扩展了一些常用语法（例如国内手机号 {{$phone.mobile}}）。你可以在“项目设置 -> Mock 设置” 里选择引擎库，默认为 Faker.js 引擎库，也是我们推荐的。

Faker.js 语法​

与原生的 Faker.js 不同，在 Apifox 中只需输入 {{$ 即可触发自动补全，轻松浏览可用的Faker.js方法，它们的对比如下：

  • Faker.js: faker.finance.accountName()
  • Apifox: {{$finance.accountName}}

基本写法​

写法说明
以{{$起始的字符串调用 Mock 语法规则生成对应的数据。
如生成的数据类型和定义的数据类型不一致，则会自动转换。
非{{$起始的字符串数据类型为string时，原样输出。
其他数据类型，会将字符串自动转换到对应的数据类型。
特殊字符：null数据类型允许为null 时，输出null。
否则自动转换，如数据类型为string，输出"null"。
特殊字符：true数据类型为boolean 时，输出true。
否则自动转换，如数据类型为string，输出"true"。
特殊字符：false数据类型为boolean 时，输出false。
否则自动转换，如数据类型为string，输出"false"。

Mock 拼接​

你可以在定义响应示例时拼接多条 Mock，以生成更丰富的数据。

假设现在需要在某一项字段中输出多个随机数据，例如预期返回某个街道与门牌号，那么可以在数据结构新增 Address 字符串并填入以下 Mock 语法： @cword()@cword()@pick(['街','路'])@integer(1,100)号。

提示

拼接多个 Mock 语法时需在结尾处添加 () 字符。
